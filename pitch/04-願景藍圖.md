# 暖暖 NuanNuan — 產品願景藍圖（給通路 / 廠商）

> **核心故事**：暖暖不是一個 App，是一個「銀髮健康守護平台」。
> 從「記錄工具」→「主動守護」→「健康生態系」，每一階段都為通路夥伴開出新的賺錢機會。
> **給夥伴的訊息**：現在加入 = 一起把這個平台做大，越早綁定越有利。

---

## 🌅 三階段成長藍圖

```
 H1 主動守護          H2 健康平台          H3 銀髮生態系
 (0-6 個月)           (6-18 個月)          (18 個月+)
 ─────────────────   ─────────────────   ─────────────────
 讓子女安心           讓健康數據有價值       成為銀髮經濟的入口
 ✅ 部分已上線        🔨 開發中            🌱 藍圖
```

---

## 🥇 Horizon 1：主動守護（0-6 個月）— 「讓子女安心」

**主題**：從「被動記錄」變「主動關心」。子女最深的恐懼是「爸媽出事我不知道」。

| 功能 | 狀態 | 廠商的機會 |
|------|------|-----------|
| **異常預警**（血壓飆高/失聯/漏藥 → 通知家人） | ✅ 已上線 | 賣點：差異化、子女付費理由 |
| **每日關懷推播**（每早一句個人化問候） | 🔨 開發中 | 提升每日開啟率 = 廣告/導購觸及 |
| **LINE 官方帳號整合**（推播到長輩最常用的 LINE） | 🔨 規劃中 | 通路可共用 LINE 觸及自己客戶 |
| **子女端儀表板**（家人視角看爸媽健康總覽） | 🔨 規劃中 | 抓住「付費決策者」 |
| **用藥交互作用警示**（多種藥 → 提醒衝突） | 🔨 規劃中 | 藥局通路超有感的賣點 |

**這一階段廠商得到**：差異化產品、子女付費入口、高黏著用戶。

---

## 🥈 Horizon 2：健康平台（6-18 個月）— 「讓數據有價值」

**主題**：累積的健康數據開始產生商業價值，從 App 升級成「平台」。

| 功能 | 廠商的機會 |
|------|-----------|
| **慢性病管理 + 回診提醒**（趨勢報告可給醫師看） | 診所通路：病患管理工具、回診率提升 |
| **藍牙裝置整合**（血壓計、血糖機自動上傳） | **賣裝置的廠商：軟硬整合、裝置綁定銷售** |
| **遠距諮詢對接**（連結藥師/營養師/醫師） | 醫療通路：諮詢分潤、加值服務 |
| **健康數據 API**（匿名趨勢報告） | **保險公司：風險評估、精準行銷** |
| **家庭多帳號方案**（一家子共用、互相照顧） | 提高客單價、家庭綁定降低流失 |

**這一階段廠商得到**：軟硬整合銷售、數據變現、加值服務分潤。

---

## 🥉 Horizon 3：銀髮生態系（18 個月+）— 「成為銀髮經濟入口」

**主題**：暖暖成為長輩生活的入口，串接銀髮經濟的各種服務。

| 功能 | 廠商的機會 |
|------|-----------|
| **保險風險評分 / 健康促進方案** | 保險公司：降低理賠率、健康獎勵機制 |
| **長照機構管理後台**（一個機構管百位長輩） | **長照通路：B2B SaaS 大單** |
| **健康電商導購**（保健品、輔具、健康餐） | **電商/品牌通路：導購分潤、精準推薦** |
| **AI 健康風險預測**（提早預警中風/跌倒風險） | 全通路：預防醫學的護城河 |
| **跨平台白標輸出**（東南亞華人市場） | 國際通路：海外市場複製 |

**這一階段廠商得到**：B2B 大單、電商分潤、預防醫學藍海、海外複製。

---

## 💰 對通路夥伴的「收入演進」

```
H1：訂閱分潤              （現在就有）
  ↓
H2：軟硬整合 + 數據授權    （裝置綁定、保險數據）
  ↓
H3：B2B SaaS + 電商導購    （長照大單、健康電商分潤）
```

> **關鍵訊息**：跟暖暖合作的收入，會隨著平台長大而**越賺越多**。早期夥伴在每個新階段都享有優先權與更好條件。

---

## 🎯 為什麼「現在」就要加入

1. **早鳥分潤更高** — 早期夥伴鎖定優惠分潤比例
2. **共同定義產品** — 你的客戶需求會影響我們開發優先順序
3. **市場卡位** — 超高齡社會明年到來，先進場先吃市場
4. **平台紅利** — 每開一個新階段，你的收入來源就多一條

>
> **重點是：每一階段都為您開新的賺錢管道。您現在加入，是綁定一個會一直長大的平台，不是一個固定的產品。**」

---

## ⚠️ 講藍圖的紀律

1. **H1 強調「已上線」** — 證明你會執行，不是空想
2. **H2/H3 講「方向」不講「日期」** — 別承諾具體上線時間
3. **每個功能都繞回「廠商賺什麼」** — 廠商只在乎自己的利益
4. **誠實分標 ✅已上線 / 🔨開發中 / 🌱藍圖** — 不要把藍圖講成已完成，會失去信任
