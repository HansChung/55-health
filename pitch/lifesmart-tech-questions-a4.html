暖暖 NuanNuan

感謝 貴公司願意提供 API。暖暖（nuan55.com）是一套 55+ 銀髮健康守護軟體，已具備異常預警、家人通知與健康儀表板。我們希望承接 貴公司環境偵測裝置的事件，做 AI 分析與家人通知。以下為串接前需釐清的技術問題，煩請逐項回覆，謝謝。
必答（決定能否開工） 重要 後續

一開放平台 / 認證

是否有開放平台 / 雲端 API 文件？可否提供連結與開發者帳號申請方式？
API 採用哪種認證？（OAuth 2.0／API Key／簽章）授權流程為何？
Token 的有效期與更新（refresh）機制？
正式與測試（sandbox）環境是否分開？base URL 各為何？
申請接入需哪些資料／審核流程？大約多久開通？

二裝置與資料

哪些裝置 SKU 提供資料 API？（人體存在、跌倒／離床、SOS、智慧床墊、門磁、溫濕度、空氣品質…）
每種裝置可取得哪些欄位？可否提供 JSON 格式範例？
數值的單位、時間戳格式與時區為何？
裝置與「使用者帳號」如何對應？一位長輩多裝置如何識別？
是否支援多租戶／多家庭的資料隔離？

三即時性與推播

取得事件是用 Webhook 推播，還是要我們輪詢（polling）？
若有 Webhook：可否推送到我們指定的 HTTPS 端點？格式為何？
緊急事件（跌倒／SOS）的延遲大約多少秒？（救援最關鍵）
若是輪詢：建議頻率？有無最小間隔限制？
事件是否保證順序與不漏送？有無重送與去重（event id）？

四用量與限制

API 的呼叫頻率限制（rate limit）為何？
單帳號可綁定的裝置數上限？
歷史資料可回查多久？是否提供歷史查詢 API？

五安全

Webhook 是否提供簽章驗證（確認事件來自 LifeSmart）？
是否支援 IP 白名單 / mTLS？
健康資料傳輸是否全程 HTTPS／加密？

六資料落地與合規 ⚠️

API 與資料伺服器機房位置在哪？（中國／海外節點）
台灣使用者資料能否落地台灣或海外節點？（涉台灣個資法，上線關鍵）
資料擁有權與授權範圍？可否存入我方資料庫做分析？
是否有資料處理協議（DPA）／隱私條款可簽署？

七測試與支援

可否提供測試帳號 + 測試裝置（或模擬器）做 POC？
是否有技術窗口／開發者社群／工單支援？
是否有 SDK（Node.js／後端）可用，還是純 REST？

八商務（技術相關）

API 介接是否收費？怎麼計（按裝置／呼叫量／月費）？
硬體由誰供應、安裝？是否有合作夥伴價格？
⭐ 若時間有限，請先回這三題（決定可行性）
  1. 是否有開放 API、用什麼認證（題 1–3）
  2. 緊急事件（跌倒／SOS）怎麼即時取得、延遲多少（題 11–13）
  3. 台灣用戶資料能否落地台灣／海外（題 22–23）
